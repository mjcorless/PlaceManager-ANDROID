Description

Recast your Android Place Library app to work with a SQLite database as its underlying model. Your app should include a minimal database in the app bundle, which is used to initialize the app's database should it not already exist in internal storage. Your app should provide the ability to add new place descriptions, to remove place descriptions, and to modify existing place descriptions. Include the same place fields/properties as defined in prior assignments: See: Single View Place Places (Android part Only), and Multi-View Place Manager.

Grading Criteria

  • 8 points. Your database definition and initialization files. Your Android app should also define and properly use a write-able database that's initialized from the app bundle to access, store, and modify place information. Your app uses Sqlite to access this database.
  • 2 points. Your app targets API 23 and uses API 23 as the minimum SDK version, it uses the current compileSdkVersion (26). These will assure it builds and runs on the grader's Android Studio (version 3) configuration. It uses the packaging constraints defined in the first assignment, linked above.
  • 5 points. Your app provides the ability to list and browse all place names, to display the details of a selected place description, to add new place descriptions, modify existing place descriptions, and delete existing place descriptions.
  • 5 points. Your app provides the capabilty for the user to view the great circle distance and initial heading between any two registered places as selected by the user.
