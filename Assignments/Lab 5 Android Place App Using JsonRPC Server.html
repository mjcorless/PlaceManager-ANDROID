Description

Create an Android app to browse and modify a collection of Place Descriptions, as you did in prior assignments. But, for this app, make network connections to a JsonRPC server to provide the model for your app, instead of storing the model on the device itself. Your app should provide the ability to browse, add, and remove places. Any changes to places or the library that are made by the app user should be pushed to the server using the JsonRPC methods defined by the server.

You may use the instructor provided JsonRPC server for places to implement your Android app: placeServerWeb.jar, or you may build your own server based on the student collection example discussed in class, and linked below.

  • studentCollectionJsonRPCServer.jar JsonRPC student collection server.
  • studentJsonRPCClientViaAsyncTask.jar JsonRPC client using AsyncTask.
  • studentJsonRPCClientViaThread.jar JsonRPC client using Thread/Handler.

Please refer to the readme.txt file in the instructor's place server, and in the student collection server for instructions for executing these servers.

Use the Place Collection JsonRPC server, to create an Android App, based on your solution to Lab3: Android App to View and Manage PlaceDescriptions. Your Android app should use the server as its only data source for obtaining place information. You should not retain a local mirror of the collection within the app, and instead use the JsonRPC methods defined by your collection to obtain model information. You may however find that you need to create a local array of place names to supply information to an adapter.

Your app should not perform network connections on the UI or Main Thread, but instead should perform these operations asynchronously. Your app should also not access view objects except on the UI or Main Thread. You are free to use either the Thread and Handler approach or AsyncTask approach to adhere to this constraint. Both approaches are shown in student collection examples linked above.

The Android app should use an entry in the file: res/values/strings.xml to record the default URL for android. Since the Android Emulator runs as a virtual machine its IP is not the same as the host/development machine, as it is with iOS simulator. Instead, your development machine is considered as the Emulator's gateway. To find this on a device or the emulator, use the Android Debugger from the command line as:

     path_to_sdk/sdk/platform-tools/adb shell
This will remotely log you onto the Emulator. Then enter the following commands to find ip's:
     ip addr show
     ip route show
Use the default gateway ip, which is: 10.0.2.2. So, your default url for the the Android app will be:

http://10.0.2.2:8080/

What To Hand-In

Structure your project as sub-directories of the folder named Assign5MyASURITEID. Use the same folder structure as used in the student collection example linked above. The sub-directories should contain the Android app, and server. If you use the instructor's server, you do not need to submit it, providing you adhere to the Json formats used by those servers.

You will submit this project, by first cleaning it to remove all generated files. Then create a jar or zip archive of the project (Assign5*) directory. You can create a Java Archive (jar) by executing the following command from a terminal in the directory which is parent to the project directory:

jar   -cvf   Assign5MyASURITEID.jar   Assign5MyASURITEID/

That archive will then be submitted via Blackboard.
